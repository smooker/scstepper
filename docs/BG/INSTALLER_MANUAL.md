# stepper_sc Ръководство за монтаж и настройка

## Първоначална настройка

### 1. Определяне на стъпки на милиметър (`spmm`)

Изчислете въз основа на микростъпките на драйвера и стъпката на водещия винт:

```
spmm = (стъпки_на_мотора_на_оборот × микростъпки) / стъпка_на_винта_мм
```

Пример: мотор 200 стъпки, микростъпки 1/8, водещ винт 4mm стъпка:
```
spmm = (200 × 8) / 4 = 400
```

Задайте и запазете:
```
set spmm 400
save
```

### 2. Определяне на инверсия на посоката (`dirinv`)

Stepper драйверът може да инвертира посоката поради оптронни входове.

1. Задайте `dirinv 0` (по подразбиране)
2. Изпълнете `mover 5` — наблюдавайте въртенето
3. Ако CW → правилно, оставете `dirinv 0`
4. Ако CCW → задайте `dirinv 1`
5. `save`

**Конвенция**: `mover` = положително = CW, `movel` = отрицателно = CCW.

### 3. Задаване на скорост при хоминг (`homespd`)

Скорост на приближаване за командата `home`. Скоростта на backoff е автоматично `homespd / 10`.

```
set homespd 1.0
save
```

Препоръчително: започнете бавно (1.0 mm/s), увеличавайте след потвърждаване надеждността на крайните изключватели.

### 4. Задаване на отстояние при хоминг (`homeoff`)

Разстояние в **стъпки** за паркиране след изключвателя след backoff.

```
set homeoff 400
save
```

При spmm=400, 400 стъпки = 1mm.

### 5. Задаване на работни скорости

| Параметър | Описание | Препоръчителна начална стойност |
|-----------|----------|--------------------------------|
| `mmpsmax` | Максимална скорост за движения и jog | 10.0 mm/s |
| `mmpsmin` | Начална/крайна скорост на рампите | 1.0 mm/s |
| `dvdtacc` | Ускорение | 50.0 mm/s² |
| `dvdtdecc` | Забавяне (може да се различава от ускорението) | 100.0 mm/s² |
| `jogmm` | Разстояние на jog бутона | 1.0 mm |
| `stepmm` | Разстояние на step бутона | 1.0 mm |

```
set mmpsmax 10
set mmpsmin 1
set dvdtacc 50
set dvdtdecc 100
set jogmm 1
set stepmm 1
save
```

### 6. Проверка на крайните изключватели

1. Изпълнете `di` (режим diag_inputs)
2. Натиснете ръчно всеки краен изключвател
3. Потвърдете, че `ES_L hit` и `ES_R hit` се появяват правилно
4. Изпълнете отново `di` за изход от диагностичен режим

### 7. Тест на хоминга

1. Позиционирайте мотора далеч от двата крайни изключвателя
2. Изпълнете `home`
3. Очакван изход:
```
home: approach ES_L @ 1.0 mm/s CCW
home: ES_L confirmed, settling...
home: backoff @ 0.10 mm/s CW
home: ES_L released, +400 steps CW
home: done
```

Ако видите `home: ABORT` — ES_L не е бил физически натиснат (фалшив EMI тригер или неправилно свързан крайен изключвател).

### 8. Проверка с Combo

Изпълнете `combo` за тест на всичките 4 рампови профила:
- Триъгълник ляво (1mm)
- Триъгълник дясно (1mm)
- Трапец ляво (5mm)
- Трапец дясно (5mm)

## Карта на EEPROM параметрите

| Адрес | Параметър | Тип | По подразбиране | Единица |
|-------|-----------|-----|-----------------|---------|
| 1 | mmpsmax | float | 50.0 | mm/s |
| 2 | mmpsmin | float | 1.0 | mm/s |
| 3 | dvdtacc | float | 100.0 | mm/s² |
| 4 | dvdtdecc | float | 80.0 | mm/s² |
| 5 | jogmm | float | 1.0 | mm |
| 6 | stepmm | float | 1.0 | mm |
| 7 | spmm | uint32 | 80 | стъпки/mm |
| 8 | dirinv | uint32 | 0 | 0=нормално, 1=инверсия |
| 9 | homespd | float | 1.0 | mm/s |
| 10 | homeoff | uint32 | 400 | стъпки |
| 11 | debug | uint32 | 0 | битово поле (bit0: подробни съобщения за бутони) |

Всички съхранени като raw `uint32_t` (floats чрез union bit-cast). Двустранично изравняване на износването в flash сектори 6 и 7 (128 KB всеки).

## Свързване

### Stepper драйвер

| Сигнал | MCU пин | Бележки |
|--------|---------|---------|
| PULSE | PB10 (TIM2_CH3) | PWM изход, ширина на импулса 50µs |
| DIR | PB14 | GPIO изход, задава се преди първия импулс |

Ако драйверът има оптронни входове (честа практика): свържете MCU пин → оптрон+ чрез 3.3V, оптрон common → GND. Задайте `dirinv 1` ако посоката е инвертирана.

### Крайни изключватели

| Сигнал | MCU пин | Бележки |
|--------|---------|---------|
| ES_L | PA3 | Активен LOW, вътрешен pull-up. Домашен изключвател |
| ES_R | PA4 | Активен LOW, вътрешен pull-up |

**Препоръчително**: добавете 4.7kΩ външен pull-up към 3.3V + 100nF кондензатор (GND) на всеки крайен изключвател за потискане на EMI. Вътрешните 40kΩ pull-up са недостатъчни в шумни среди (соларни инвертори, stepper драйвери).

### Бутони

| Сигнал | MCU пин | Бележки |
|--------|---------|---------|
| JOG L | PA6 | Активен LOW, вътрешен pull-up |
| JOG R | PA7 | Активен LOW, вътрешен pull-up |
| STEP L | PB0 | Активен LOW, вътрешен pull-up |
| STEP R | PB1 | Активен LOW, вътрешен pull-up |

Същата препоръка: външни pull-up + кондензатори за среди с много EMI.

### Зумер

| Сигнал | MCU пин | Бележки |
|--------|---------|---------|
| BUZZ | PB15 | Активен LOW — самоосцилиращ зумер |

### Debug / USB

| Сигнал | MCU пин | Бележки |
|--------|---------|---------|
| USB D- | PA11 | USB CDC виртуален COM порт |
| USB D+ | PA12 | USB CDC виртуален COM порт |
| SWDIO | PA13 | SWD debug (Black Magic Probe) |
| SWCLK | PA14 | SWD debug (Black Magic Probe) |
| LED | PC13 | Heartbeat (превключване 500ms) |

## Съображения за EMI

Соларните инвертори и stepper мотор драйверите генерират комутационен шум, който се свързва с дълги неекранирани кабели. Симптоми:

- Фалшиви тригери на крайни изключватели (`ES_L hit` / `ES_R hit` без физическо натискане)
- Фантомни натискания на бутони (ghost jog/step движения)
- Самодиагностиката при стартиране заседва в CQ цикъл

**Хардуерно решение (PCB)**: 4.7kΩ pull-up към 3.3V + 100nF кондензатор към GND на всеки входен пин.

**Софтуерни мерки** (вече имплементирани):
- Самодиагностика при стартиране с детекция на задържан вход
- ES_L е изключен от самодиагностиката (моторът може да е паркиран върху домашния изключвател)
- Команди `buttons off` / `endstops off`
- Хомингът използва дебаунсвано GPIO polling (не EXTI)
- 30ms edge lockout на крайните изключватели в нормален режим
- Блокиране по посока от крайните изключватели
- Jog release: незабавно ISR-ниво спиране + 50ms заключване срещу bounce

## Отстраняване на неизправности

| Симптом | Причина | Решение |
|---------|---------|---------|
| Моторът не се движи | Захранването на драйвера е изключено | Включете захранването на мотор драйвера |
| `mover` върви CCW | DIR е инвертиран | `set dirinv 1` след това `save` |
| Стартирането заседва в CQ цикъл | Вход задържан в LOW (EMI или окабеляване) | Оправете окабеляването, добавете pull-up или изчакайте след залез |
| `home` ABORT | EMI предизвиква фалшив ES_L | Ще се реши с външни pull-up на PCB |
| Странни символи на терминала | Init стрингове от minicom | Фиксирано: RX буферът се изчиства преди main loop |
| Грешен крайен изключвател при `home` | ES_L/ES_R кабелите са разменени | Разменете кабелите на конектора |
