# stepper_sc Фърмуер

STM32F411CEU6 (Black Pill) контролер за stepper мотор с USB CDC интерфейс за команди, трапецовиден скоростен профил и EEPROM емулация във вградената flash памет.

## Управление на мотора (stepper.c)

Автомат на състоянията: IDLE → ACCEL → CONST → DECEL → IDLE
Асиметрични рампи: `dvdtacc` и `dvdtdecc` могат да се различават за независими профили на ускорение/забавяне.

- PWM: TIM2 канал 3 @ 96 MHz, ширина на импулса ~50 µs (5000 тика)
- Рампови таблици: предварително изчислени чрез sqrt(v² + 2·a·s) с хардуерен VSQRT
- ISR: обновява TIM2 ARR (период) при всяка стъпка за плавен скоростен профил
- Посока: PB14 (DIR пин), задава се преди първия импулс
- Инверсия на посоката: параметърът `dirinv` XOR-ва DIR изхода за оптронни драйвери
- Проследяване на позицията: брояч `posSteps` в ISR, абсолютна позиция след хоминг
- `posHomed`: флаг, задава се след успешен хоминг; изисква се от `range` и `moveto`
- `rangeUsableMm`: записва се от `range` (raw − homeoff); изисква се от `moveto`

## USB CDC Команди

| Команда              | Описание                        |
|----------------------|---------------------------------|
| move \<mm\>          | Движение по разстояние          |
| movel \<mm\>         | Движение ляво (отрицателно)     |
| mover \<mm\>         | Движение дясно (положително)    |
| steps \<n\>          | Движение с брой стъпки          |
| set \<param\> \<val\>| Задаване на параметър           |
| stop                 | Принудително забавяне и спиране |
| params               | Принтира текущите параметри     |
| save                 | Запис на параметрите в EEPROM   |
| dump                 | Debug dump на промеливи         |
| cls                  | Изчиства екрана на терминала    |
| uptime               | Принтира HAL_GetTick() ms       |
| reset                | Софтуерен рестарт (NVIC_SystemReset) |
| combo                | 4 тестови движения (2 триъгълни + 2 трапецни) |
| help                 | Списък с команди                |
| home                 | Хоминг: намиране на ES_L, backoff, паркиране |
| range                | Измерва разстоянието между крайните изключватели; изисква posHomed |
| moveto \<mm\>        | Абсолютно позициониране; изисква home + range |
| morse \<текст\>      | Морзе код (неблокиращо)         |
| buttons on/off       | Включване/изключване на EXTI за бутоните |
| endstops on/off      | Включване/изключване на EXTI за крайните изключватели |
| diag_inputs (di)     | Превключване на диагностичен режим на входовете |
| diag_outputs (do)    | PULSE+DIR тест (парола: motorola) |

Поддържат се escape последователности на терминала (стрелки, F1-F4).

## EEPROM Емулация (eeprom_emul_uint32_t.c)

Двустранично изравняване на износването с flash сектори 6 и 7 (128 KB всеки).

- 8-байтови атомарни записи: 4-байтово заглавие (виртуален адрес) + 4-байтови данни
- Състояния на страница: ERASED (0xFFFFFFFF) → RECEIVE (0xEEEEEEEE) → VALID (0xAAAAAAAA)
- Компакция при смяна на страница когато активната страница се запълни
- Защита при загуба на захранване: незавършени записи се игнорират при възстановяване
- Виртуални адреси 1-11 съответстват на 11-те параметъра на мотора

## Конфигурация на часовника

- HSE: 24 MHz
- PLL: M=12, N=96 → 96 MHz SYSCLK
- APB1: 48 MHz (÷2)
- APB2: 96 MHz (÷1)
- TIM2: 96 MHz, начален период 9599, импулс 4800

## Билд

```bash
cd firmware
make          # създава build/stepper_sc.elf/.hex/.bin
```

След регенерация от CubeMX: изпълнете `./post_cubemx.sh` за възстановяване на -Wno-error=unused-parameter.

## Debug

```bash
./go_gdb.sh   # стартира gdb-dashboard + script2.gdb
```

GDB команди:
- `ag` — свързване с Black Magic Probe
- `ld` — зареждане на stepper_sc.elf/hex + проверка

SVD инспекция на регистри чрез PyCortexMDebug + STM32F411.svd.
